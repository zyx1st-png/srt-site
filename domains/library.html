领域与专题 · Domain Library

五个领域，一张不抹平差异的阅读地图

这页不是文件名清单，也不是把五个领域都说成同一个理论。它把 SRT-Pub 中的 owner 主文、Compact Core、实验／硬化材料与 Annex 接口分开摆放：先知道材料在系统中的位置，再决定读哪一层。

读法：主文 → Compact Core → 证据／硬化 → Annex。 Registry 与 README 是导航，Annex 是接口；它们不替代 Core 的 canonical 定义。
桥接不等于证明。 AI、神经、物理、哲学与灵性材料的 claim-status、proxy 和 open tension 必须随材料一起阅读。
一条总线：所有领域都在问“什么被保留、什么被排开、结果如何写回”，但每个领域的承重对象不同。物理侧重可达性与热力学边界，神经侧重机制与代理测量，AI 侧重架构状态与主体门槛，哲学侧重解释与规范，灵性侧重方向、回返与实践问责。
01 · AI

架构状态、合成主体与意识门槛

AI 不是 SRT 的定义发动机，而是最尖锐的 boundary-test：它迫使我们把能力、报告、持续性、代价与后果回流拆开。当前材料的核心不是“AI 是否已经有意识”，而是“什么条件必须先被说明，强断言才有资格出现”。

主线：本体与危机 owner

从 SRT_AI_00_Crisis.md 的人工系统危机入口，到 SRT_AI_01_Ontology.md 的 intelligence／consciousness 区分，再到 SRT_AI_02_Mortality_Wisdom.md 的有限性与智慧，主线始终把“会输出”与“有 stakes”分开。

  • 先读 AI_POSITIONING_NOTE.md，确定架构态、赌注与越权边界。
  • 再读 SRT_AI_Claim_Status.md，区分 bridge、proxy、操作性 rubric 与公开简写。
进入 AI 公开入口 →

三条 Annex 支线 interface

Architecture_Annex 处理工程接口、AGI program themes 与 reservoir computing；Consciousness_Annex 处理 controlled hallucination、report／reality、moral status 与 bearer-unit；Ontology_Annex 处理 active-inference override、passive-recording fallacy、RTC 与 suffering-condition。

  • 相似的内部动力学不自动带来 subjecthood。
  • 外部报告、规划能力与模型规模都不能单独承重。

开放问题 re-assess

当前推理态、无持久记忆、无具身的 LLM 只能给出架构态限定的低直接赌注判断；一旦出现持久记忆、具身行动、不可逆后果或稳定的后果回流，必须重新评估，而不是沿用旧结论。

  • 能力 ≠ 关切；报告 ≠ 现实锚定；复杂度 ≠ 可支付摩擦。
  • SRT_AI_Consciousness_Evaluation_Rubric.md 是评估工具，不是 consciousness 定义。

从神经选择到意识机制，再到可检验的代理

神经科学区是 SRT 最接近实验的领域之一，但“接近实验”不等于“已经证明本体”。这里要把 owner 机制、clinical proxy、hardening draft、实验设计与 Annex 外部理论比较分层处理。

机制主线 compact core

SRT_Neural_Mechanisms_CompactCore.md 承接神经选择、G_hat_theta、Ψf、d-value 与 L1→L2 沉积；SRT_Consciousness_Mechanisms_CompactCore.md 则把意识写成稳定的 concern-weighted L1 anchoring。两者是当前公开阅读的最短正式路径。

  • 主文：SRT_Neural_Mechanisms.md、SRT_Consciousness_Mechanisms.md。
  • 守门：SRT_Neuroscience_Claim_Status.md 与 SRT_Neuro_Axioms_Claim_Status.md。

读者路线 · READING ORDER

按问题进入，而不是按文件数量迷路

想理解一个领域

先读对应公开页，再读本页的 owner／Compact Core 卡片，最后看该领域的 claim-status。这样能知道“它说了什么”与“它有权说到哪里”为何不是同一个问题。

想做跨域比较

先回到 跨域显影 看形式骨架，再进入 理论对照 看机制差异，最后沿本页的 Annex 与实验入口追溯支持材料。

想判断一条强断言

查它的 owner、P-level、proxy 与 open tension；如果只在 Annex、hardening draft 或 Operations 记录中出现，就不能把它当成 canonical 定义或已证实定律。

材料边界：本页的文件名与路线来自 SRT-Pub 的当前目录、README、compact registry、claim-status 与 Annex 索引。它们是公众导览与检索地图，不把仓库中的桥接、实验、硬化或伴随文本升级为 canonical 理论。具体公式、阈值与定义仍应回到 SRT-Pub 的 canonical anchors 核对。

从一个领域进入，再沿回链横向阅读

领域之间共享问题骨架，但不共享同一套机制、证据或责任边界。
